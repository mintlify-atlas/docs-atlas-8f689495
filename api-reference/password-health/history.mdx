---
title: 'POST /public/teams/PasswordHealthHistory'
description: 'Returns historical password health snapshots ordered by date, enabling you to track security improvements and measure the impact of remediation initiatives over time.'
---

The `PasswordHealthHistory` endpoint returns a time-series of password health snapshots for your team. Each snapshot captures the same aggregate metrics as the current report — total passwords, safe, weak, reused, compromised, and average strength — recorded at a point in the past. Use this data to build trend charts, measure whether security initiatives had a meaningful effect, or satisfy audit requirements.

## Endpoint

```
POST https://api.dashlane.com/public/teams/PasswordHealthHistory
```

## Authentication

All requests require a `Bearer` token in the `Authorization` header. Your API key follows the format `DLP_<teamUuid>_<accessKey>_<secretKey>`. See [API keys](/authentication/api-keys) for instructions on generating keys.

## Request

Send an empty JSON object as the request body. No parameters are required.

<ParamField body="body" type="object">
  Empty JSON object. No fields are accepted.
</ParamField>

## Response

<ResponseField name="history" type="array" required>
  Array of password health snapshots ordered by date. Each element represents the team's aggregate password health on a specific date.

  <Expandable title="properties">
    <ResponseField name="date" type="string" required>
      The date of the snapshot in `YYYY-MM-DD` format.
    </ResponseField>

    <ResponseField name="passwordsCount" type="number" required>
      Total number of passwords stored across all team members on this date.
    </ResponseField>

    <ResponseField name="safePasswords" type="number" required>
      Number of passwords that were considered safe on this date.
    </ResponseField>

    <ResponseField name="weakPasswords" type="number" required>
      Number of passwords that did not meet minimum strength requirements on this date.
    </ResponseField>

    <ResponseField name="reusedPasswords" type="number" required>
      Number of passwords that were reused across a member's vault on this date.
    </ResponseField>

    <ResponseField name="compromisedPasswords" type="number" required>
      Number of passwords that had appeared in known data breaches on this date.
    </ResponseField>

    <ResponseField name="averagePasswordStrength" type="number" required>
      Team-wide average password strength score on this date, from `0` to `100`.
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  Use the history data to build trend charts or track whether a security initiative — such as a company-wide password reset campaign — produced a measurable improvement in your team's scores.
</Note>

<Tip>
  Combine this endpoint with `PasswordHealthReport` in a single dashboard: display the latest snapshot from the report as the "current state" card, then plot the history array to show the trend line below it.
</Tip>

## Example

### Request

```bash
curl --request POST \
  --url https://api.dashlane.com/public/teams/PasswordHealthHistory \
  --header 'Authorization: Bearer DLP_teamUuid_accessKey_secretKey' \
  --header 'Content-Type: application/json' \
  --data '{}'
```
