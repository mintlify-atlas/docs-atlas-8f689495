The Dashlane Public API is a read-only REST API for Dashlane team accounts. Every endpoint accepts a JSON request body and returns a JSON response. All requests use the `POST` method — there are no `GET`, `PUT`, or `DELETE` operations because the API does not modify any data.

## Base URL

All endpoints share a single base URL:

```
https://api.dashlane.com
```

## Required headers

Every request must include two headers:

| Header | Value |
|---|---|
| `Content-Type` | `application/json` |
| `Authorization` | `Bearer <your-api-key>` |

Your API key follows the format `DLP_<teamUuid>_<accessKey>_<secretKey>`. See the [API keys guide](/authentication/api-keys) for instructions on generating and managing keys.

```bash
curl --request POST \
  --url https://api.dashlane.com/public/teams/Members \
  --header 'Content-Type: application/json' \
  --header 'Authorization: Bearer DLP_teamUuid_accessKey_secretKey' \
  --data '{}'
```

## Available endpoints

The API exposes six endpoints across three resource groups. All paths are relative to `https://api.dashlane.com`.

| Endpoint | Description |
|---|---|
| `POST /public/teams/Members` | List all members of the team with their status and role |
| `POST /public/teams/MemberDevices` | Retrieve devices registered to team members |
| `POST /public/teams/PasswordHealthReport` | Get the current password health snapshot for the team |
| `POST /public/teams/PasswordHealthHistory` | Retrieve historical password health metrics over time |
| `POST /public/teams/TeamInfo` | Fetch team account information and seat usage |
| `POST /public/teams/TeamPolicies` | Read the security policies configured for the team |

## Error handling

All errors are returned as JSON with an HTTP status code. See the [error codes reference](/api-reference/errors) for a full list of status codes, their causes, and how to resolve them.
