title: "Get started with the Dashlane Public API"
description: "Generate an API key, authenticate with a Bearer token, and make your first request to the Dashlane Public API — all in under five minutes."
---

This guide walks you through everything you need to make your first call to the Dashlane Public API. You will install the Dashlane CLI, generate an API key, and retrieve a list of your team members using `curl`. The entire process takes about five minutes.

<Info>
  You must be a **Dashlane team admin** to generate API keys. If you don't have admin access, contact your Dashlane administrator.
</Info>

<Steps>
  <Step title="Install the Dashlane CLI">
    The Dashlane CLI (`dcli`) is used to generate and manage API keys. Follow the [installation instructions](https://cli.dashlane.com/install) for your operating system.

    After installation, verify it works:

    ```bash
    dcli --version
    ```
  </Step>

  <Step title="Log in with the CLI">
    Authenticate the CLI with your Dashlane account. Follow the [authentication guide](https://cli.dashlane.com/personal/authentication) if you need help with this step.

    ```bash
    dcli login
    ```

    Log in with your Dashlane admin credentials when prompted.
  </Step>

  <Step title="Generate an API key">
    Create an API key for the Dashlane Public API. Pass a description that identifies the key's purpose — you'll see this in the key list later.

    ```bash
    dcli team public-api create-key "My key"
    ```

    The CLI outputs your API key in this format:

    ```
    Bearer DLP_teamUuid_accessKey_secretKey
    ```

    <Warning>
      Copy this key immediately. It is shown **only once** and cannot be retrieved again. Store it in a secrets manager or environment variable — never in source code. The `DLP_` prefix makes it easy to detect accidental exposure when scanning codebases.
    </Warning>

    You can also list existing keys or revoke a key when it's no longer needed:

    ```bash
    dcli team public-api list-keys
    dcli team public-api revoke-key <accessKey>
    ```
  </Step>

  <Step title="Make your first API request">
    Use `curl` to call the Members endpoint. Replace the placeholder with the key you just generated.

    ```bash
    curl -X POST 'https://api.dashlane.com/public/teams/Members' \
      -H 'Content-Type: application/json' \
      -H 'Authorization: Bearer DLP_teamUuid_accessKey_secretKey' \
      -d '{}'
    ```

    A successful response returns a JSON object with a `members` array:

    ```json
    {
      "members": [
        {
          "login": "alice@example.com",
          "status": "accepted",
          "hasDesktopApp": true,
          "isBiometricsEnabled": false,
          "accountCreationDateUnix": 1690000000,
          "lastUpdateDateUnix": 1712000000,
          "devices": 2
        },
        {
          "login": "bob@example.com",
          "status": "accepted",
          "hasDesktopApp": true,
          "isBiometricsEnabled": true,
          "accountCreationDateUnix": 1695000000,
          "lastUpdateDateUnix": 1713500000,
          "devices": 1
        }
      ]
    }
    ```

    <Tip>
      You can also import the [OpenAPI 3.0 specification](https://get.dashlane.com/public-api/openapi.json) into tools like Postman or Insomnia to explore all endpoints with an interactive playground.
    </Tip>
  </Step>
</Steps>
